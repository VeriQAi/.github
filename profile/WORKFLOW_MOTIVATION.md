# Why VeriQAi Exists

Assessment in engineering courses involves a real tension: the tools that make grading efficient (structured rubrics, automated scoring) tend to create friction for students and overhead for instructors, while the tools that are easy to use (free-form submissions, flexible formatting) make consistent, scalable grading difficult.

We built GradeBridge because the existing options forced an unnecessary choice: invest many hours setting up rigid submission templates to unlock AI-assisted grading, or skip templates entirely and grade everything by hand. Neither option is sustainable in a large course. GradeBridge structures the submission workflow (from assignment generation through student submission and Gradescope autograding) so that you get consistent, automatically-scored submissions without asking students to jump through formatting hoops, and without sending any student data through a third-party server.

The MATLAB Grader Problem Generator exists for a simpler reason: writing the four artifacts that make up a single MATLAB Grader problem (description, reference solution, learner template, and test cases) takes an experienced instructor an hour or more. The test case API alone has enough edge cases to slow anyone down. The generator eliminates that authoring burden so instructors can focus on calibrating the problem to their learning objectives rather than wrestling with syntax.

Both tools bring your own API key and run entirely in the browser. The goal is to make high-quality AI-assisted assessment accessible without requiring institutional procurement, platform lock-in, or trust that a vendor will handle student data responsibly.
